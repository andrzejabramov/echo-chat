# 📄 Спецификация проекта: «Эхо-чат с поддержкой геолокации»

## 1. Общее описание

Веб-приложение для двустороннего обмена сообщениями через протокол WebSocket с реализацией эхо-механизма и функцией отправки текущей геолокации. Проект выполнен на чистом JavaScript (ES6+), без использования фреймворков и сборщиков, с соблюдением принципов модульной архитектуры и разделения ответственности.

## 2. Архитектура и структура

Проект построен по модульному принципу. Каждый модуль отвечает за конкретную зону ответственности:
```

src/
├── index.html # Точка входа, семантическая разметка
├── css/styles.css # Стилизация (CSS Variables, Flexbox, адаптивность)
└── js/
├── main.js # Композиция модулей, DI, инициализация
├── core/WebSocketClient.js # Инкапсуляция WS-соединения, автопереподключение
├── services/
│ ├── ChatService.js # Бизнес-логика, фильтрация эхо-ответов
│ └── GeoService.js # Работа с Geolocation API + сетевой фолбэк
├── ui/ChatUI.js # Управление DOM, рендеринг, обработка событий
└── utils/
├── dom.js # Хелперы для безопасной работы с DOM
└── url.js # Валидация координат, генерация OSM-ссылок

````

## 3. Технологический стек
| Компонент | Технология / API |
|-----------|------------------|
| Клиентская логика | Vanilla JavaScript (ES6 Modules) |
| Сетевой обмен | `WebSocket` (`wss://`) |
| Геолокация | `navigator.geolocation` (W3C Geolocation API) |
| Карты | OpenStreetMap (параметры `?mlat=&mlon=#map=`) |
| Стилизация | CSS3 (Custom Properties, Flexbox, `prefers-reduced-motion`) |
| Запуск | `python3 -m http.server` (или любой статический сервер) |

## 4. Функциональные требования
| Требование | Реализация |
|------------|-----------|
| ✅ Ввод текста → отправка → отображение своего сообщения | Мгновенный рендер типа `sent` перед отправкой в WS |
| ✅ Получение эхо от сервера и вывод в чат | Обработка события `message`, рендер типа `echo` |
| ✅ Отправка геолокации по кнопке | Запрос `getCurrentPosition()` → генерация ссылки OSM → рендер типа `geo` |
| ✅ Игнорирование эхо-ответа на геолокацию | Флаг `_isAwaitingGeoEcho` в `ChatService` + проверка JSON-структуры |
| ✅ Индикация статуса соединения | Динамическое обновление UI при `open`/`close`/`error` |
| ✅ Автопереподключение при разрыве | Экспоненциальная задержка, лимит 10 попыток в `WebSocketClient` |
| ✅ Поддержка клавиатуры | Отправка по `Enter`, фокус на поле ввода, `aria-*` атрибуты |

## 5. Внешние сервисы и API
| Сервис | Назначение | URL |
|--------|-----------|-----|
| Эхо-сервер | Приём и возврат отправленных данных | `wss://ws.postman-echo.com/raw` |
| Геолокация (нативная) | Получение координат устройства | `navigator.geolocation` |
| Геолокация (фолбэк) | Определение по IP при таймауте на десктопе | `https://ipapi.co/json/` |
| Карты | Визуализация координат | `https://www.openstreetmap.org/` |

> ⚠️ Примечание: оригинальный сервер `wss://echo-ws-service.herokuapp.com` недоступен с конца 2022 года. В проекте используется стабильная альтернатива Postman Echo.

## 6. Инструкция по запуску и проверке
### Запуск локально
```bash
# 1. Клонировать репозиторий
git clone https://github.com/andrzejabramov/echo-chat.git && cd echo-chat

# 2. Запустить статический сервер
python3 -m http.server 8000 --directory src

# 3. Открыть в браузере
#    → http://localhost:8000
````

> 🔒 Важно: проект **не работает** при открытии через `file://` из-за ограничений безопасности WebSocket. Требуется HTTP/HTTPS сервер.

### Проверка функционала (чек-лист)

1. Статус в шапке меняется на `Подключено ✓`
2. Ввод текста + `Enter` → появляется синее сообщение, затем серое эхо
3. Нажатие `📍 Геолокация` → запрос разрешения → появление зелёной ссылки на карту
4. При отправке геолокации эхо от сервера **не отображается**
5. При отключении интернета статус меняется на `Отключено ◌`, соединение восстанавливается автоматически

## 7. Обработка ошибок и безопасность

| Сценарий                      | Поведение системы                                                  |
| ----------------------------- | ------------------------------------------------------------------ |
| Разрыв WebSocket              | Автопереподключение с логированием, UI-индикатор                   |
| Отказ в доступе к геолокации  | Чёткое сообщение с инструкцией по проверке настроек                |
| Таймаут определения координат | Прозрачный фолбэк на IP-геолокацию с пометкой «Приблизительно»     |
| Ввод спецсимволов в сообщения | Безопасный рендеринг через `textContent` / экранирование           |
| CSP-политика                  | Настроена только для доверенных источников (`'self'`, `wss://...`) |

## 8. Ограничения и возможные улучшения

| Ограничение                            | Причина                        | Будущее улучшение                                    |
| -------------------------------------- | ------------------------------ | ---------------------------------------------------- |
| Фолбэк по IP зависит от стороннего API | Бесплатные тарифы имеют лимиты | Вынести на свой бэкенд или использовать `ip-api.com` |
| Нет сохранения истории                 | Stateless-архитектура          | Добавить `localStorage` с экспортом в `.txt`         |
| Нет WebSocket-аутентификации           | Учебный формат                 | Реализовать JWT или token-based handshake            |
| Адаптив только до `480px`              | Базовая mobile-first верстка   | Добавить tablet-breakpoint, тёмную тему              |
